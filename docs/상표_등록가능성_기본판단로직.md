# 상표 등록가능성 기본판단 로직 v1.0

## 1. 문서 목적
이 문서는 상표 등록가능성 판단 프로그램의 **기본 판단 순서와 기준**을 정의한다.  
향후 프로그램 수정 시 이 문서를 **기본 로직 기준 문서(Source of Truth)** 로 사용한다.

이 문서는 다음 두 방향을 동시에 만족하도록 설계한다.

- 기존 프로그램의 강점인 **발음/명칭 유사도 판단, 문자열 유사도, 유사군코드 관련 판단**은 유지한다.
- 다만 판단 순서를 바로잡아, **상품 유사성 필터를 먼저 적용한 뒤** 표장 유사성과 혼동 가능성을 판단하도록 한다.

---

## 2. 핵심 원칙

### 원칙 1. 상품 유사성 필터가 먼저다
선행상표와의 충돌 판단은 반드시 다음 순서로 한다.

1. 지정상품/서비스의 류(Class)와 유사군코드를 먼저 확정한다.
2. 그 기준으로 선행상표 후보를 필터링한다.
3. 그 다음에만 발음, 외관, 관념 유사도를 평가한다.
4. 마지막으로 혼동 가능성을 종합한다.

즉, **전혀 다른 상품영역의 선행상표는 원칙적으로 점수에 크게 반영하면 안 된다.**

### 원칙 2. 같은 류라고 무조건 유사상품은 아니다
같은 류(Class)에 속하더라도 상품의 성질, 용도, 유통경로, 수요자가 다르면 유사성이 약할 수 있다.

### 원칙 3. 다른 류라고 무조건 무관한 것도 아니다
류가 다르더라도 아래 경우는 예외적으로 검토한다.

- 유사군코드가 같은 경우
- 경제적 견련성이 강한 경우
- 주지·저명상표인 경우
- 수요자 기만 가능성이 있는 경우

### 원칙 4. 유사군코드가 상품 유사성의 1차 기준이다
상품 유사성은 우선적으로 유사군코드 기준으로 판단한다.

- 동일 유사군코드: 가장 강한 충돌 후보
- 동일 류, 다른 유사군코드: 보조 검토 후보
- 다른 류, 다른 코드: 원칙적으로 제외
- 다른 류라도 동일 코드 또는 경제적 견련성이 있으면 예외 검토

### 원칙 5. 절대적 거절사유와 상대적 거절사유를 분리한다
- 절대적 거절사유: 식별력 여부(상표법 제33조)
- 상대적 거절사유: 선행상표 충돌 여부(상표법 제34조)

두 판단은 분리하여 수행하고, 최종 결과에서 다시 종합한다.

## 3. 기본 판단 절차

## 3-1. 1단계 - 입력 정규화
입력값을 아래 구조로 정리한다.

- 상표명
- 상표 유형(문자 / 문자+로고 / 로고)
- 조어 여부
- 선택한 류(Class)
- 구체 상품명 또는 서비스명
- 선택한 유사군코드 전체
- KIPRIS 선행상표 검색 결과

정규화 목표는 단순 문자열 비교가 아니라, **상표 + 지정상품 + 유사군코드 문맥**을 함께 확보하는 것이다.

## 3-2. 2단계 - 식별력(절대적 거절사유) 판단
선행상표와 비교하기 전에, 입력 상표 자체가 식별력을 가지는지 판단한다.

판단 항목은 다음과 같다.

1. 보통명칭
2. 관용표장
3. 성질표시
4. 현저한 지리적 명칭
5. 흔한 성 또는 명칭
6. 간단하고 흔한 표장
7. 기타 식별력 없음

이 단계에서는 결과를 점수보다 먼저 **사유 리스트**로 정리한다.

예시:
- 제33조 제1항 제3호 가능성 있음 - 성질표시
- 제33조 제1항 제6호 가능성 있음 - 간단하고 흔한 표장
- 사용에 의한 식별력 보완 가능성 있음

중요:
식별력은 **지정상품과의 관계 속에서** 판단해야 한다.  
같은 단어라도 어떤 상품에서는 성질표시가 되지만, 다른 상품에서는 식별력을 가질 수 있다.

## 3-3. 3단계 - 선행상표 후보 필터링
선행상표 전체를 곧바로 점수에 넣지 말고, 아래 3단계로 필터링한다.

실제 구현에서는 선행상표 후보를 아래 4개 그룹으로 분리한다.

- `group_exact_code`: 동일 유사군코드
- `group_same_class`: 동일 류이지만 유사군코드는 다름
- `group_related_market`: 타 류이지만 경제적 견련성, 동일 유사군코드, 저명성, 기만 우려 등으로 예외 검토가 필요한 경우
- `group_irrelevant`: 타 류, 타 코드, 경제적 견련성이 없어 원칙적으로 점수 계산에서 제외되는 경우

이 중 `group_irrelevant`는 검색 건수에는 포함할 수 있으나, `top_prior`, 상대적 거절사유 판단, 최종 감점에는 반영하지 않는다.

### 1차 우선군: 동일 유사군코드
- 사용자가 선택한 유사군코드와 동일한 선행상표
- 류가 다르더라도 동일 유사군코드면 우선 검토
- 가장 강한 충돌 후보

### 2차 보조군: 동일 류, 다른 유사군코드
- 같은 류지만 코드가 다른 경우
- 자동 충돌로 보지 않고 보조 검토군으로 유지
- 상품 성질, 용도, 유통경로가 유사한지 추가 판단

### 3차 예외군: 다른 류, 다른 코드
원칙적으로 제외한다.  
다만 아래 경우에는 후보로 남긴다.

- 경제적 견련성이 강한 경우
- 소비자가 동일 출처로 오인할 가능성이 큰 경우
- 주지·저명상표인 경우
- 품질 오인, 수요자 기만, 부정한 목적 문제가 있는 경우

**중요 규칙:**  
3류를 선택했는데 25류 선행상표가 있다는 이유만으로 자동 감점하거나 0%로 판단하면 안 된다.  
3류와 25류가 실질적으로 유사한 상품군인지, 같은 유사군코드인지, 또는 예외 사정이 있는지를 먼저 판단해야 한다.

## 3-4. 4단계 - 표장 유사성 판단
상품 유사성 필터를 통과한 후보에 대해서만 표장 유사성을 판단한다.

판단 요소는 다음 3가지다.

- 외관 유사성
- 호칭/발음/명칭 유사성
- 관념 유사성

기본 원칙:
- 문자상표는 호칭 비중이 높다.
- 문자+로고는 호칭과 외관을 함께 본다.
- 로고 중심 상표는 외관 비중을 높인다.
- 결합상표는 전체관찰을 기본으로 하되, 필요하면 요부도 보조 검토한다.

### 유지해야 할 사항
기존 프로그램이 가진 다음 로직은 유지한다.

- 발음/명칭에 의한 유사도 판단
- 기존 문자열 유사도 판단
- 기존 similarity_percent 또는 유사 기능
- 기존 선행상표 검색 결과 활용 방식

단, 이 로직들은 **상품 유사성 필터를 통과한 후보에 우선 적용**되어야 한다.

## 3-5. 5단계 - 상품 유사성 판단
표장 유사성과 별도로 상품 유사성을 독립 평가한다.

기본 기준:
- 동일 유사군코드: 매우 높음
- 동일 류 + 인접 상품군: 높음
- 동일 류 + 비인접 상품군: 중간 또는 낮음
- 타 류 + 동일 유사군코드: 높음
- 타 류 + 경제적 견련성 강함: 중간
- 타 류 + 견련성 없음: 제외 또는 0

여기서 상품 유사성은 다음 요소를 함께 본다.

- 품질/형상
- 용도
- 생산 부문
- 판매 채널
- 수요자 범위
- 거래 현실

## 3-6. 6단계 - 혼동 가능성 종합판단
최종적으로 다음 3축을 종합한다.

- 표장 유사성
- 상품 유사성
- 실제 거래에서의 혼동 가능성

기본 판정 기준:

### 고위험
- 동일 또는 매우 유사한 유사군코드
- 발음/외관/관념 유사도가 높음
- 소비자층과 거래 경로가 겹침

### 중위험
- 같은 류 또는 경제적 견련성 있음
- 표장 유사도가 중간 이상
- 조정 여지 존재

### 저위험
- 상품 유사성이 낮음
- 표장 유사도가 일부 있어도 출처 혼동 가능성이 낮음

### 충돌 제외
- 상품 유사성이 없음
- 경제적 견련성 없음
- 저명상표 예외도 없음

**중요 규칙:**  
상품 유사성이 낮거나 없으면, 표장 유사성만으로 치명적 결론을 내리지 않는다.

## 3-7. 7단계 - 예외 보정
다음 항목은 별도의 예외 보정으로 처리한다.

### 사용에 의한 식별력
원래 식별력이 약한 상표라도 장기간 사용으로 출처식별력을 얻은 경우 예외 가능성을 안내한다.

### 주지·저명상표
상품이 다르더라도 저명상표에 편승하거나 수요자 기만 우려가 있으면 검토 범위를 확장한다.

### 부정한 목적 출원
자동 확정 판단보다는 별도 경고 또는 주의 항목으로 표시한다.

## 4. 최종 출력 구조

최종 결과는 아래 3개 블록으로 보여준다.

### A. 식별력 판단
- 식별력 문제 없음
- 식별력 약함
- 거절 가능성 큼
- 사용에 의한 보완 가능성 있음

### B. 선행상표 충돌 판단
- 동일 유사군코드 충돌
- 동일 류 충돌
- 경제적 견련성 충돌
- 충돌 후보 없음

### C. 종합 등록 가능성
- 높음
- 보통
- 주의 필요
- 낮음
- 매우 낮음

### D. 검색 결과와 충돌 후보의 구분
결과 화면과 보고서에는 아래 수치를 함께 보여준다.

- 전체 검색 건수
- 상품 유사성 필터 통과 건수
- 실제 충돌 위험 건수
- 제외된 후보 건수

또한 아래 설명을 함께 제공한다.

- 왜 일부 검색 결과가 상품 유사성 검토에서 제외되었는지
- 왜 필터 통과 선행상표가 0건이면 점수가 높게 형성되는지
- 왜 식별력 약함이 있더라도 충돌 후보가 없으면 등록 가능성이 여전히 높을 수 있는지

## 5. 점수화 원칙

### 원칙 1. 점수는 최종 종합값이다
점수는 상품 유사성 필터를 통과한 후보만 반영한다.

### 원칙 2. 무관한 상품군은 강한 감점 대상이 아니다
다른 류, 다른 유사군코드, 경제적 견련성 없음의 경우에는 원칙적으로 강한 감점을 주지 않는다.

### 원칙 3. 0% 또는 등록 불가 판단은 엄격하게 제한한다
다음과 같은 경우에만 0% 또는 등록 불가에 가까운 평가를 할 수 있다.

- 식별력 자체가 매우 약함
- 동일 또는 매우 유사한 유사군코드에서
- 발음/외관/관념이 매우 유사한 선행상표가 존재하고
- 혼동 가능성이 높음

즉, **전혀 다른 상품영역의 상표가 있다는 이유만으로 0%가 되면 안 된다.**

### 원칙 4. 검색 결과와 실제 충돌 후보를 구분한다
검색 결과가 존재한다는 사정만으로 점수를 낮추지 않는다.  
반드시 상품 유사성 필터를 통과한 후보만 상대적 거절사유와 최종 점수에 반영한다.

### 원칙 5. 점수 캘리브레이션 기준
아래 범위를 기본 캘리브레이션 기준으로 사용한다.

- 식별력 보통 이상 + 필터 통과 선행상표 0건: 82~90
- 조어상표 + 필터 통과 선행상표 0건: 88~95
- 식별력 약함 + 필터 통과 선행상표 0건: 72~82
- 동일 유사군코드 충돌 후보 존재 + 표장 유사도 높음: 50 이하 가능
- 동일 류이나 비유사 코드 + 표장 유사도 중간: 55~75
- 타 류/타 코드 후보만 존재하고 필터 통과 후보 0건: 원칙적으로 70 미만으로 떨어뜨리지 않음

즉, “선행상표 없음”이 아니라 **“상품 유사성 검토를 통과한 선행상표 없음”** 을 기준으로 상대적 거절사유 리스크를 본다.

### 원칙 6. 판매업 코드는 기본적으로 중립이다
`S` 코드가 선택되었다는 이유만으로 자동 감점하지 않는다.  
동일 판매업 코드에서 실제 충돌 후보가 있는 경우에만 제한적으로 반영한다. 판매업 코드는 보호 범위 확대 목적의 지정일 수 있으므로 기본적으로 중립 또는 약한 주의 신호로 본다.

### 원칙 7. 타 류/타 코드 후보 제외 원칙
타 류이면서 타 코드이고 경제적 견련성도 없는 후보는 `group_irrelevant`로 분류한다.  
이 그룹은 전체 검색 건수 설명에는 포함할 수 있지만, 최종 점수와 주요 충돌 후보 목록에서는 제외한다.

### 실제 예시 1. Lexai / 20류 가구 vs 9류 소프트웨어
`Lexai`가 20류 가구에 출원되고, 검색 결과는 일부 존재하지만 상품 유사성 필터를 통과한 선행상표가 0건이라면 상대적 거절사유 리스크는 매우 낮게 본다.  
반면 동일한 `Lexai`가 9류 소프트웨어에 출원되고 동일 또는 유사한 선행상표가 존재하면, 같은 표장이라도 20류보다 의미 있게 낮은 점수를 부여한다.

### 실제 예시 2. 3류 화장품 vs 25류 의류
3류 화장품을 선택했는데 25류 의류 선행상표만 검색된 경우, 동일 유사군코드나 경제적 견련성이 없으면 `group_irrelevant`로 본다.  
이 경우 검색 결과는 있을 수 있으나 최종 감점에는 거의 반영하지 않는다.

## 6. 구현 가이드

### 유지할 것
- 기존 발음/명칭 유사도 로직
- 기존 문자열 유사도 로직
- 기존 유사군코드 추천 로직
- 기존 KIPRIS 검색 흐름
- 기존 Streamlit UI 흐름

### 바꿔야 할 것
- 선행상표 결과를 곧바로 점수화하지 말고 먼저 상품 유사성 필터를 적용할 것
- 상품 유사성이 없는 후보는 점수 반영을 최소화하거나 제외할 것
- 절대적 거절사유와 상대적 거절사유를 분리할 것
- 결과 화면에서 “왜 위험한지”를 식별력/상품유사성/표장유사성 관점에서 나눠 설명할 것
- 기존 유사도 로직, 발음 로직, 유사군코드 로직은 유지하되 최종 점수 합성 단계에서 상품 유사성 필터를 우선 반영할 것

## 7. 한 줄 기준 문장
향후 구현은 아래 한 문장을 기준으로 삼는다.

**먼저 지정상품의 류와 유사군코드를 기준으로 선행상표 후보를 필터링하고, 그 후에만 외관·호칭·관념의 유사도를 평가하며, 마지막으로 상품의 유사성과 거래 현실을 종합하여 소비자 혼동 가능성을 판단한다. 단, 타 류이더라도 동일 유사군코드, 경제적 견련성, 주지·저명성, 기만 우려가 있으면 예외적으로 검토 대상에 포함한다.**
